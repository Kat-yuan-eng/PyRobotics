# Localization -- 定位

本模块实现了3种定位算法，覆盖从经典扩展卡尔曼滤波到粒子滤波再到多传感器融合的完整定位方案，用于在运动模型预测与传感器观测之间融合出最优位姿估计。

## 模块列表

| 算法 | 文件 | 核心函数 | 状态维度 | 观测模型 | 适用场景 |
|------|------|---------|---------|---------|---------|
| 扩展卡尔曼滤波 | ekf_localizer.py | `ekf_localize()` | 4维 [x,y,yaw,v] | GPS位置 | 弱非线性、高斯噪声 |
| 粒子滤波 | pf_localizer.py | `pf_localize()` | 4维 [x,y,yaw,v] | 路标距离+方位 | 强非线性/非高斯 |
| 协方差交叉融合 | fusion_localizer.py | `fusion_localize()` | 4维 [x,y,yaw,v] | GPS+路标 | 多传感器互补融合 |

---

## 算法详细说明

### 1. 扩展卡尔曼滤波 (ekf_localizer.py)

EKF 是卡尔曼滤波在非线性系统上的经典推广。预测步用运动模型雅可比 `jacob_motion()` 传播协方差，更新步用观测矩阵 `jH` 计算卡尔曼增益，实现预测与观测的加权融合。

运动模型（CV模型）：

```
x_{k+1} = x_k + v_k * dt * cos(yaw_k)
y_{k+1} = y_k + v_k * dt * sin(yaw_k)
yaw_{k+1} = yaw_k + omega * dt
v_{k+1} = v_k
```

观测模型：`z = H @ x`，直接观测 x, y 位置。特色功能为自适应过程噪声 `adaptive_Q()`：当加速度或角速度较大时自动增大 Q 矩阵，使滤波器在急转弯/急加速时更信任观测而非预测，避免滤波发散。

关键参数：`Q`（过程噪声协方差）、`R`（观测噪声协方差）、`accel_scale`（自适应Q的加速度缩放因子）。EKF 计算量最小，但要求非线性足够温和且噪声近似高斯。

### 2. 粒子滤波 (pf_localizer.py)

用 `n_particles` 个带权粒子近似状态后验分布。预测步对每个粒子施加含噪声的控制输入，传播运动模型；更新步根据粒子预测位置与路标的距离/方位和实际观测之差，计算高斯似然更新权重；当有效粒子数 `N_eff = 1/(Σw²)` 低于阈值时触发系统重采样（`systematic_resample`）。

观测模型为路标距离+方位角：`z_i = [d, θ]`，其中 `d = ||x - landmark_i||`，`θ = atan2(dy, dx) - yaw`。系统重采样在累积权重上等间距采样，比随机重采样方差更低。`pf_estimate()` 和 `pf_covariance()` 分别输出加权均值和加权协方差。

关键参数：`n_particles`（粒子数，越多精度越高但计算量越大）、`R_motion`（运动噪声）、`Q_obs`（观测噪声）、`n_threshold`（重采样阈值）。PF 的核心优势在于对非高斯、多模态分布的建模能力，代价是计算量与粒子数成正比。

### 3. 协方差交叉融合 (fusion_localizer.py)

将 EKF（处理 GPS 观测）和 PF（处理路标观测）的估计结果进行协方差交叉融合（Covariance Intersection）。融合公式：`P_fused = (P_ekf^{-1} + P_pf^{-1})^{-1}`，`x_fused = P_fused @ (P_ekf^{-1} @ x_ekf + P_pf^{-1} @ x_pf)`，即以各自协方差逆矩阵为权重的加权平均。协方差越小（估计越确定）的滤波器在融合中权重越大。

该融合策略不需要假设两个估计之间的相关性（保守融合），即使 EKF 和 PF 使用了共同的运动模型预测（存在相关性），融合结果仍然一致。在 GPS 信号良好时 EKF 主导，在路标密集时 PF 主导，两者互补。

关键参数：通过 EKF 和 PF 的参数间接控制。融合本身无额外超参数。

## 算法对比总表

| 维度 | EKF | PF | 融合 |
|------|-----|-----|------|
| 方法类型 | 参数化（单高斯） | 非参数化（粒子集） | 混合融合 |
| 状态表示 | 均值+协方差 | N个带权粒子 | 融合均值+协方差 |
| 是否需要雅可比 | 是 | 否 | 间接需要(EKF部分) |
| 计算复杂度 | O(n³) | O(N_particles × N_landmarks) | O(n³) + O(N×L) + O(n³) |
| 非高斯处理 | 不支持 | 支持 | 部分支持 |
| 多模态分布 | 不支持 | 支持 | 部分支持 |
| 全局定位 | 否（需初值） | 是（均匀初始化） | 视初始化方式 |
| 传感器 | GPS | 路标距离+方位 | GPS + 路标 |

## 可视化

所有算法模块支持 `show_animation = True` 交互式可视化，遵循 PythonRobotics 动画规范：

- 动画方式：`plt.cla()` + `plt.pause(0.001)` 逐帧刷新
- 颜色语义：红(.r)=参考路径、蓝(-b)=实际轨迹、绿(xg)=目标点、黑(.k)=障碍物
- 运行方式：直接执行各模块脚本，如 `python Localization/ekf_localizer.py`

### 模块动画说明

| 模块 | 动画内容 |
|------|---------|
| EKF | 真实轨迹(蓝)+航位推算(黑)+EKF估计(红)对比动画，GPS观测点可视化 |
| PF | 粒子云动画，粒子分布+真实轨迹+估计轨迹 |
| Fusion | EKF+PF+Fusion三方法对比动画 |

## 模块间关联关系

```
GPS观测 ──────→ ekf_localizer ──→ (x_ekf, P_ekf) ──┐
                                                     ├──→ fusion_localizer ──→ (x_fused, P_fused)
路标观测 ─────→ pf_localizer ───→ (x_pf, P_pf) ────┘
```

EKF 和 PF 各自独立运行，分别处理不同传感器的观测。`fusion_localizer` 汇聚两者的估计结果，通过协方差交叉融合输出最终位姿。三个模块共享相同的运动模型（CV模型），但观测模型不同：EKF 用 GPS 直接位置观测，PF 用路标距离+方位观测。
